Why Recall is the Primary Metric

False Negative = Bank approves a defaulter
→ Average loss: ~60% of loan value

False Positive = Bank rejects a good customer
→ Average loss: ~5% missed interest

Catching defaulters is 12× more important than avoiding false rejections. Recall measures exactly this.
Key Design Decisions
⚖️
SMOTE, Not Random Oversampling
Generates realistic synthetic defaulters using k-nearest neighbors in feature space. Applied only to training data — test set mirrors real-world distribution.
🧮
Engineered Financial Ratios
Raw columns aren't enough. DTI ratio, credit utilisation, installment-to-income, and loan-to-income capture the true financial burden on a borrower.
⚖️
scale_pos_weight in XGBoost
Even after SMOTE, XGBoost's scale_pos_weight=4 penalises false negatives more heavily during gradient descent training.
🔍
SHAP for Legal Compliance
Banks legally cannot deny a loan with only "AI said so." SHAP provides mathematically grounded, feature-level explanations that satisfy ECOA / GDPR requirements.
💰
Financial Cost-Benefit Analysis
Translates ML metrics into real dollar values. Shows examiners exactly how much money the model saves vs. the cost of false alarms.
🚀
Streamlit Over Flask/React
Same visual result in 1/5th the code. One command to run. Examiners can interact with live predictions on any laptop without setup.

Quick Start
Get running in 2 commands
pip install -r requirements.txt
streamlit run app.py
Then open http://localhost:8501 in your browser. Click "Generate Demo Data" to start immediately — no Kaggle download needed.

For the real dataset: download accepted_2007_to_2018Q4.csv from kaggle.com/datasets/wordsforthewise/lending-club and upload via the app.

2

5
